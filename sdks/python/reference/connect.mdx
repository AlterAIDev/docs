## `connect()`

Run headless OAuth from a CLI or notebook. Opens a browser, polls until the user finishes.

### Signature

```python
async def connect(
    *,
    providers: list[str | Provider] | None = None,
    grant_policy: GrantPolicyInput | None = None,
    timeout: float = 300.0,
    poll_interval: float = 2.0,
    open_browser: bool = True,
) -> list[ConnectResult]
```

### Parameters

| Name | Default | Notes |
|---|---|---|
| `providers` | `None` | Restrict the picker. `None` = any allowed by the app config. |
| `grant_policy` | `None` | Set max TTL on the resulting grant. |
| `timeout` | `300.0` | Seconds to poll before raising `ConnectTimeoutError`. |
| `poll_interval` | `2.0` | Seconds between polls. |
| `open_browser` | `True` | Set `False` over SSH; the SDK prints the URL to copy. |

### Returns

`list[ConnectResult]` — one per connected account. Each item carries `grant_id`, `provider_id`, `account_identifier`, `scopes`.

### Raises

`ConnectFlowError`, `ConnectDeniedError`, `ConnectTimeoutError`, `ConnectConfigError`.

### Example

```python
results = await app.connect(providers=["github"])
print(results[0].grant_id)
```

## `authenticate()`

Open the IDP's sign-in flow and return the resulting JWT.

### Signature

```python
async def authenticate(*, timeout: float = 300.0) -> AuthResult
```

### Returns

`AuthResult` with `user_token` and `user_info`.

### Example

```python
auth = await app.authenticate()
print("user:", auth.user_info, "jwt:", auth.user_token)
```

Requires the app's IDP to be configured for OIDC sign-in. See [OAuth → Identity Providers](/oauth/identity-providers/overview).

## `create_connect_session()`

Build a connect URL backend-side; redirect the user there or hand it to a frontend popup SDK.

### Signature

```python
async def create_connect_session(
    *,
    allowed_providers: list[str | Provider] | None = None,
    return_url: str | None = None,
    allowed_origin: str | None = None,
    metadata: dict[str, str] | None = None,
    grant_policy: GrantPolicyInput | None = None,
) -> ConnectSession
```

### Returns

`ConnectSession` with `session_token` and `connect_url`. Pass `session_token` to the [Connect.js SDK](/sdks/connect-js/quickstart) or redirect to `connect_url`.

### Example

```python
session = await app.create_connect_session(
    allowed_providers=["google", "slack"],
    return_url="https://app.example.com/oauth/done",
)
return RedirectResponse(session.connect_url)
```

## `create_managed_secret_connect_session()`

Mint a Connect session for the user → agent delegation flow on a [managed
secret](/managed-secrets/overview). The user opens `connect_url` in a
browser and consents; the backend writes a delegation row authorizing
the named agent to USE the user's existing managed-secret access.

The delegation does NOT create new access — it lets an agent borrow
the user's existing direct or group-derived access. If the user has no
eligible access, the Connect UI shows a clear message and the
delegation is not created.

### Signature

```python
async def create_managed_secret_connect_session(
    *,
    template_slug: str,
    delegated_agent_id: str,
    user_token: str,
    requested_ttl_seconds: int | None = None,
    delegated_agent_name: str | None = None,
    allowed_origin: str | None = None,
    return_url: str | None = None,
) -> ManagedSecretConnectSession
```

### Parameters

| Name | Default | Notes |
|---|---|---|
| `template_slug` | required | Canonical template identifier (kebab-case). The SDK rejects other shapes at the boundary. |
| `delegated_agent_id` | required | UUID of the agent being authorized. Must be active in the calling app. |
| `user_token` | required | IDP JWT identifying the consenting user. Required — there is no system-principal fallback for managed-secret delegation. |
| `requested_ttl_seconds` | `None` | Caller-suggested lifetime. Effective TTL = `min(requested, template max, source expiry, 90-day default)`. |
| `delegated_agent_name` | `None` | Display name for the consent screen. Defaults to the agent's stored display name. |
| `allowed_origin` | `None` | Origin for postMessage in the popup flow. |
| `return_url` | `None` | Return URL for the mobile redirect flow. |

### Returns

`ManagedSecretConnectSession` with `session_token`, `connect_url`,
`expires_in`, `expires_at`. The user is taken to `connect_url`; after
they approve, the agent calls `app.request(grant_id=...)` using the
**delegation id** returned via postMessage or the `return_url` query
string.

### Raises

`AlterValueError` for client-side validation failures (empty values,
non-kebab template slug, non-positive TTL).

`BackendError` for backend rejections — template not found, agent not
eligible, user JWT invalid, etc. The status code is collapsed to a
single value per failure family to avoid leaking which internal check
rejected the request.

### Example

```python
# Resolve the agent's UUID once at startup, then reuse on each call.
billing_bot = await app.agents.get_by_name("billing-bot")

session = await app.create_managed_secret_connect_session(
    template_slug="stripe-api-key",
    delegated_agent_id=billing_bot.id,
    user_token=user_jwt,
    requested_ttl_seconds=30 * 86400,
)
return RedirectResponse(session.connect_url)
```

## `verify_user_token()`

Verify a JWT against the configured IDP without making a provider call.

### Signature

```python
async def verify_user_token(token: str) -> str | None
```

### Returns

The verified `sub` claim, or `None` if invalid.

### Example

```python
sub = await app.verify_user_token(jwt)
if sub is None:
    raise HTTPException(401, "invalid token")
```
