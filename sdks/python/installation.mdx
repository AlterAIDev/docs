The Alter SDK for Python provides secure credential management with automatic token injection for OAuth, API keys, and custom credentials, plus audit logging, connection management, and AI agent tracking.

## Requirements

- Python 3.11 or higher
- `pip` or `poetry` package manager

## Installation

<Tabs>
  <Tab title="pip">
    ```bash
    pip install alter-sdk
    ```
  </Tab>

  <Tab title="Poetry">
    ```bash
    poetry add alter-sdk
    ```
  </Tab>
</Tabs>

## Verify Installation

```python
import alter_sdk
print(alter_sdk.__version__)  # v0.4.0+
```

## Dependencies

The SDK has minimal dependencies:

- `httpx` - Async HTTP client with HTTP/2 and connection pooling
- `pydantic` - Data validation and settings management

No provider-specific dependencies are needed. The SDK works with **any** OAuth provider through a single generic `vault.request()` method. All HTTP methods are supported: GET, POST, PUT, PATCH, DELETE, HEAD, and OPTIONS.

## Architecture Notes

<Note>
  Your API key is **never sent to external provider APIs**. The SDK ensures complete credential isolation — your API key only communicates with Alter Vault, while provider requests are made separately without it. All backend requests are signed with HMAC-SHA256 using a derived signing key for request integrity, authenticity, and replay protection.
</Note>

<Warning>
  The SDK **does not cache tokens locally** by design. This ensures:
  - Real-time policy enforcement on every request
  - Instant revocation when users disconnect
  - Complete audit trail for compliance (always enabled, including per-request tracking IDs)
</Warning>

<Note>
  The SDK enforces a **closed instance guard** — after `close()` is called (or the context manager exits), any further `request()` calls raise `AlterSDKError`. `close()` is idempotent and safe to call multiple times.
</Note>

<Note>
  The SDK provides a `TimeoutError` (subclass of `NetworkError`) for timeout-specific handling. Catch `TimeoutError` before `NetworkError` to implement retry strategies for timed-out requests.
</Note>

<Note>
  All input validation errors (invalid `api_key`, invalid/missing `actor_type`, missing `actor_identifier`, invalid URL scheme, missing `path_params`) raise `AlterSDKError` — the base exception class for the SDK. The `actor_type` (use `ActorType` enum) and `actor_identifier` parameters are **required** at initialization.
</Note>
