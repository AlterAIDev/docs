The Alter SDK for TypeScript provides secure credential management with automatic token injection for OAuth, API keys, and custom credentials, plus connection management, audit logging, and AI agent tracking.

## Requirements

- Node.js 18 or higher
- TypeScript 5.0+ (recommended)

## Installation

<Tabs>
  <Tab title="npm">
    ```bash
    npm install @alter-ai/alter-sdk
    ```
  </Tab>

  <Tab title="pnpm">
    ```bash
    pnpm add @alter-ai/alter-sdk
    ```
  </Tab>

  <Tab title="yarn">
    ```bash
    yarn add @alter-ai/alter-sdk
    ```
  </Tab>
</Tabs>

## Verify Installation

```typescript
import { AlterVault } from "@alter-ai/alter-sdk";
console.log("Alter SDK loaded successfully"); // v0.4.0+
```

## Dependencies

The SDK has **zero runtime dependencies** -it uses the built-in `fetch` API (Node.js 18+).

No provider-specific dependencies are needed. The SDK works with **any** OAuth provider through a single generic `vault.request()` method.

## Architecture Notes

<Note>
  Your API key is **never sent to external provider APIs**. The SDK ensures complete credential isolation — your API key only communicates with Alter Vault, while provider requests are made separately without it. All backend requests are cryptographically secured for request integrity, authenticity, and replay protection.
</Note>

<Warning>
  The SDK **does not cache tokens locally** by design. This ensures:
  - Real-time policy enforcement on every request
  - Instant revocation when users disconnect
  - Complete audit trail for compliance (always enabled)
</Warning>

<Note>
  The SDK enforces a **closed instance guard** -after `close()` is called, any further `request()` calls throw `AlterSDKError`. `close()` is idempotent and safe to call multiple times.
</Note>

<Note>
  The SDK provides a `TimeoutError` (subclass of `NetworkError`) for timeout-specific handling. Catch `TimeoutError` before `NetworkError` to implement retry strategies for timed-out requests.
</Note>

<Note>
  All input validation errors (invalid `apiKey`, invalid/missing `actorType`, missing `actorIdentifier`, invalid URL scheme, missing `pathParams`) throw `AlterSDKError` -the base error class for the SDK. The `actorType` (use `ActorType` enum) and `actorIdentifier` parameters are **required** at initialization.
</Note>
