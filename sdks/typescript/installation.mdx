The Alter SDK for TypeScript provides secure OAuth token management with automatic token injection, audit logging, and AI agent tracking.

## Requirements

- Node.js 18 or higher
- TypeScript 5.0+ (recommended)

## Installation

<Tabs>
  <Tab title="npm">
    ```bash
    npm install @alter-ai/alter-sdk
    ```
  </Tab>

  <Tab title="pnpm">
    ```bash
    pnpm add @alter-ai/alter-sdk
    ```
  </Tab>

  <Tab title="yarn">
    ```bash
    yarn add @alter-ai/alter-sdk
    ```
  </Tab>
</Tabs>

## Verify Installation

```typescript
import { AlterVault } from "@alter-ai/alter-sdk";
console.log("Alter SDK loaded successfully");
```

## Dependencies

The SDK has **zero runtime dependencies** — it uses the built-in `fetch` API (Node.js 18+).

No provider-specific dependencies are needed. The SDK works with **any** OAuth provider through a single generic `vault.request()` method.

## Architecture Notes

<Note>
  The SDK uses **two separate HTTP clients** for security:
  1. **Alter client** — Communicates with Alter Vault backend (includes your API key)
  2. **Provider client** — Calls external APIs like Google, GitHub, Slack (no API key, prevents credential leakage)
</Note>

<Warning>
  The SDK **does not cache tokens locally** by design. This ensures:
  - Real-time policy enforcement on every request
  - Instant revocation when users disconnect
  - Complete audit trail for compliance
  - Maximum security with no local token storage (tokens stored in module-private WeakMap with hardened fetch capture)

  Backend Redis caching keeps response times under 10ms.
</Warning>

<Note>
  The SDK includes **SSRF prevention** with case-insensitive URL scheme validation, blocking `file://`, `ftp://`, and mixed-case bypass attempts. Only `https://` and `http://` schemes are allowed. SSRF violations throw `AlterSDKError`.
</Note>

<Note>
  All input validation errors (invalid `apiKey`, invalid `actorType`, SSRF URL scheme checks, missing `pathParams`) throw `AlterSDKError` — the base error class for the SDK.
</Note>
