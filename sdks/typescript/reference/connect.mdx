## `connect()`

Run headless OAuth from a CLI or script. Opens a browser, polls until the user finishes.

### Signature

```typescript
async connect(options: {
  providers?: Array<string | Provider>;
  grantPolicy?: GrantPolicyInput;
  timeout?: number;
  pollInterval?: number;
  openBrowser?: boolean;
}): Promise<ConnectResult[]>
```

### Parameters

| Name | Default | Notes |
|---|---|---|
| `providers` | `undefined` | Restrict the picker. |
| `grantPolicy` | `undefined` | Set max TTL on the resulting grant. |
| `timeout` | `300000` | ms to poll before throwing `ConnectTimeoutError`. |
| `pollInterval` | `2000` | ms between polls. |
| `openBrowser` | `true` | Set `false` over SSH; SDK prints the URL. |

### Returns

`ConnectResult[]` — one per connected account.

### Throws

`ConnectFlowError`, `ConnectDeniedError`, `ConnectTimeoutError`, `ConnectConfigError`.

### Example

```typescript
const results = await app.connect({ providers: ["github"] });
console.log(results[0].grantId);
```

## `authenticate()`

Open the IDP's sign-in flow and return the resulting JWT.

### Signature

```typescript
async authenticate(options?: { timeout?: number }): Promise<AuthResult>
```

### Returns

`AuthResult` with `userToken` and `userInfo`.

### Example

```typescript
const auth = await app.authenticate();
console.log("user:", auth.userInfo, "jwt:", auth.userToken);
```

Requires the app's IDP to be configured for OIDC sign-in. See [OAuth → Identity Providers](/oauth/identity-providers/overview).

## `createConnectSession()`

Build a connect URL backend-side; redirect the user there or hand it to a frontend popup SDK.

### Signature

```typescript
async createConnectSession(options?: {
  allowedProviders?: Array<string | Provider>;
  returnUrl?: string;
  allowedOrigin?: string;
  metadata?: { ipAddress?: string; userAgent?: string };
  grantPolicy?: { maxTtlSeconds?: number; defaultTtlSeconds?: number };
  requiredScopes?: Record<string, string[]>;
  agent?: string;
}): Promise<ConnectSession>
```

`requiredScopes` lets a session ask for a narrower per-provider scope set than the app's Dev Portal default (must be a subset of the Dev Portal allowlist). `agent` delegates the resulting OAuth grant to a managed agent — when combined with user consent at the Connect UI, the OAuth grant is created under the user's principal and the named agent is granted access to it.

### Returns

`ConnectSession` with `sessionToken` and `connectUrl`. Pass `sessionToken` to the [Connect.js SDK](/sdks/connect-js/quickstart) or redirect to `connectUrl`.

### Example

```typescript
const session = await app.createConnectSession({
  allowedProviders: ["google", "slack"],
  returnUrl: "https://app.example.com/oauth/done",
});
res.redirect(session.connectUrl);
```

## `createManagedSecretConnectSession()`

Mint a Connect session for the user → agent delegation flow on a
[managed secret](/managed-secrets/overview). The user opens
`connectUrl` in a browser and consents; the backend writes a
delegation row authorizing the named agent to USE the user's existing
managed-secret access.

The delegation does NOT create new access — it lets an agent borrow
the user's existing direct or group-derived access. If the user has
no eligible access, the Connect UI shows a clear message and the
delegation is not created.

### Signature

```typescript
async createManagedSecretConnectSession(
  options: {
    templateSlug: string;
    delegatedAgentId: string;
    userToken: string;
    requestedTtlSeconds?: number;
    delegatedAgentName?: string;
    allowedOrigin?: string;
    returnUrl?: string;
  },
): Promise<ManagedSecretConnectSession>
```

### Parameters

| Name | Default | Notes |
|---|---|---|
| `templateSlug` | required | Canonical template identifier (kebab-case). The SDK rejects other shapes at the boundary. |
| `delegatedAgentId` | required | UUID of the agent being authorized. Must be active in the calling app. |
| `userToken` | required | IDP JWT identifying the consenting user. Required — there is no system-principal fallback for managed-secret delegation. |
| `requestedTtlSeconds` | `undefined` | Caller-suggested lifetime. Effective TTL = `min(requested, template max, source expiry, 90-day default)`. |
| `delegatedAgentName` | `undefined` | Display name for the consent screen. Defaults to the agent's stored display name. |
| `allowedOrigin` | `undefined` | Origin for postMessage in the popup flow. |
| `returnUrl` | `undefined` | Return URL for the mobile redirect flow. |

### Returns

`ManagedSecretConnectSession` with `sessionToken`, `connectUrl`,
`expiresIn`, `expiresAt`. The user is taken to `connectUrl`; after
they approve, the agent calls `app.request({ grantId: ... })` using
the **delegation id** returned via postMessage or the `returnUrl`
query string.

### Throws

`AlterValueError` for client-side validation failures (empty values,
non-kebab template slug, non-positive TTL).

`BackendError` for backend rejections — template not found, agent not
eligible, user JWT invalid, etc. The status code is collapsed to a
single value per failure family to avoid leaking which internal check
rejected the request.

### Example

```typescript
// Resolve the agent's UUID once at startup, then reuse on each call.
const billingBot = await app.agents.getByName("billing-bot");

const session = await app.createManagedSecretConnectSession({
  templateSlug: "stripe-api-key",
  delegatedAgentId: billingBot.id,
  userToken: userJwt,
  requestedTtlSeconds: 30 * 86400,
});
res.redirect(session.connectUrl);
```

## `verifyUserToken()`

Verify a JWT against the configured IDP without making a provider call.

### Signature

```typescript
async verifyUserToken(token: string): Promise<string | null>
```

### Returns

The verified `sub` claim, or `null` if invalid.

### Example

```typescript
const sub = await app.verifyUserToken(jwt);
if (sub === null) throw new Error("invalid token");
```
