title: 'alter sdk-passthrough'
description: 'Make a one-off authenticated provider request through a grant.'
---

`sdk-passthrough` is an escape hatch for making a single authenticated request to a provider's API through a grant — the kind of thing you'd otherwise do with the runtime SDK's `request()`. It exists for debugging ("fetch a token for this grant and hit the provider once to see what comes back"), deliberately namespaced so it reads as *"I'm doing something unusual"* rather than blending in with the management commands.

```
alter sdk-passthrough request <grant-id> --url <url> [--method ...] [--body ...] [--header ...]
```

<Note>
This is the one CLI command that does **not** use the PAT. It signs the request with a **runtime API key** — pass it via the `ALTER_API_KEY` environment variable (preferred) or `--api-key`. The PAT from `alter auth login` does not work here. Mint a runtime key with [`alter keys mint`](/reference/cli/commands/keys#mint).

For anything beyond one-off debugging, use the [Python](/reference/python-sdk) or [TypeScript](/reference/typescript-sdk) SDK in application code.
</Note>

## request

Make a single authenticated call against a grant. The grant must be active and owned by the calling app.

| Flag | Required | Description |
|---|---|---|
| `--url <url>` | yes | The full provider URL to call (e.g. `https://www.googleapis.com/oauth2/v3/userinfo`). Must be `http://` or `https://`. There is no relative-path resolution — pass the complete URL. |
| `--method <method>` | no | HTTP method (default `GET`). One of `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `OPTIONS`. |
| `--body <json>` | no | JSON request body, passed verbatim to the provider. |
| `--header <kv>` | no | Extra header in `Name: value` form. Repeatable. Reserved headers (the `Authorization` header and Alter's own request headers) are rejected — Alter injects the credential for you. |
| `--api-key <key>` | no | Runtime API key, if not using `ALTER_API_KEY`. Accepts any runtime key minted from the dashboard or CLI — `alter_rk_…`, `alter_ak_…`, `alter_dk_…`, `alter_pk_…`, or the legacy `alter_key_…`. PAT plaintext (`alter_pat_…`) is rejected — use `alter auth login` for that. Inline values print a shell-history warning. |

The response body goes to **stdout** (pipe it into `jq`); the status line and response headers go to **stderr**, so the body stays clean for downstream tools.

```bash
export ALTER_API_KEY="alter_rk_..."

alter sdk-passthrough request <grant-id> \
  --url https://www.googleapis.com/oauth2/v3/userinfo \
  | jq .

# With a body
alter sdk-passthrough request <grant-id> \
  --method POST \
  --url https://slack.com/api/chat.postMessage \
  --body '{"channel":"#general","text":"hello"}'
```

## Recipes

**Debug a grant from scratch**

```bash
# 1. Mint a runtime key (printed once) and export it.
alter keys mint --name debug --scopes grants:read --output json
export ALTER_API_KEY="alter_rk_..."
# 2. Call the provider through the grant and inspect the response.
alter sdk-passthrough request <grant-id> --url https://www.googleapis.com/oauth2/v3/userinfo | jq .
```
