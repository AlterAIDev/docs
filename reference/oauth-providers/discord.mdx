## Overview

Connect your users to Discord for identity verification, server membership data, and messaging integrations.

| Property | Value |
|----------|-------|
| **Provider ID** | `discord` |
| **Category** | Communication |
| **PKCE** | Supported |
| **Token refresh** | Automatic |
| **Redirect URI** | Shown in [Developer Portal](https://portal.alterauth.com) |

## Step 1: Create a Discord OAuth App

<Steps>
  <Step title="Go to the Discord Developer Portal">
    Navigate to [discord.com/developers/applications](https://discord.com/developers/applications) and sign in.
  </Step>

  <Step title="Create a new application">
    Click **New Application**, enter a name, and click **Create**.
  </Step>

  <Step title="Configure OAuth2">
    In the left sidebar, click **OAuth2**. Under **Redirects**, add the Alter callback URL from the [Developer Portal](https://portal.alterauth.com).
  </Step>

  <Step title="Get credentials">
    Copy the **Client ID** and **Client Secret** from the OAuth2 page.
  </Step>
</Steps>

<Warning>
  Never share your Client Secret publicly. Store it securely and only enter it in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterauth.com](https://portal.alterauth.com) and navigate to the application.
  </Step>

  <Step title="Add Discord provider">
    Go to **OAuth Providers** > **Add Provider** > **Discord**.
  </Step>

  <Step title="Enter credentials">
    - **Client ID**: Paste your Discord Client ID
    - **Client Secret**: Paste your Discord Client Secret
  </Step>

  <Step title="Select scopes">
    Choose the scopes the application needs.
  </Step>

  <Step title="Save">
    Click **Save**. The provider is now active.
  </Step>
</Steps>

## Available Scopes

| Scope | Description |
|-------|-------------|
| `identify` | View the user's username, avatar, and banner |
| `email` | View the user's email address |
| `guilds` | View the servers the user is a member of |
| `guilds.join` | Add the user to a server |
| `guilds.members.read` | View the user's member info in servers |
| `messages.read` | Read messages from all client channels (local RPC server API access) |
| `bot` | Add a bot to a server (requires bot user) |
| `connections` | View the user's linked third-party accounts |
| `webhook.incoming` | Create incoming webhooks in servers |

## Notes

- The `messages.read` scope is part of the local RPC server API, which is in private beta and requires Discord approval.
- For most user-facing integrations, `identify` and `email` are sufficient.
- See the [Discord OAuth2 documentation](https://docs.discord.com/developers/topics/oauth2) for more details.
