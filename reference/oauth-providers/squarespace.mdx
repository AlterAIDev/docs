## Overview

Connect your users to Squarespace for website content management, ecommerce orders, inventory, and product data.

| Property | Value |
|----------|-------|
| **Provider ID** | `squarespace` |
| **Category** | E-Commerce |
| **PKCE** | Supported |
| **Token refresh** | Automatic |
| **Redirect URI** | Shown in [Developer Portal](https://portal.alterauth.com) |

## Step 1: Register a Squarespace OAuth Client

<Steps>
  <Step title="Open the Squarespace OAuth documentation">
    Navigate to the [Squarespace OAuth guide](https://developers.squarespace.com/commerce-apis/oauth).
  </Step>

  <Step title="Submit the OAuth client registration form">
    Squarespace OAuth clients are registered by submitting the request form linked under **Register the client and obtain OAuth 2.0 credentials**.
  </Step>

  <Step title="Provide the redirect URI">
    Provide the client name, icon, and the Alter callback URL from the [Developer Portal](https://portal.alterauth.com) as a **Redirect URI**.
  </Step>

  <Step title="Get credentials">
    Squarespace reviews the registration and returns the **Client ID** and **Client Secret**.
  </Step>
</Steps>

<Warning>
  Never share your Client Secret publicly. Store it securely and only enter it in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterauth.com](https://portal.alterauth.com) and navigate to the application.
  </Step>

  <Step title="Add Squarespace provider">
    Go to **OAuth Providers** > **Add Provider** > **Squarespace**.
  </Step>

  <Step title="Enter credentials">
    - **Client ID**: Paste your Squarespace Client ID
    - **Client Secret**: Paste your Squarespace Client Secret
  </Step>

  <Step title="Select scopes">
    Choose the scopes the application needs.
  </Step>

  <Step title="Save">
    Click **Save**. The provider is now active.
  </Step>
</Steps>

## Available Scopes

| Scope | Description |
|-------|-------------|
| `website.orders.read` | View order and fulfillment information |
| `website.orders` | Send order data and mark orders as fulfilled |
| `website.transactions.read` | Access transactional order and donation data |
| `website.inventory.read` | View inventory stock levels |
| `website.inventory` | View and update inventory stock levels |
| `website.products.read` | View product information |
| `website.products` | View product information and modify products |
| `website.contacts.read` | View customer contact information and address book entries |
| `website.contacts` | View, create, update, and delete contacts and address book entries |

## Notes

- Scopes are comma-separated when requested.
- Access tokens expire after 30 minutes. Alter Vault handles automatic token refresh; refresh tokens last 7 days and are rotated (single-use) on each refresh.
- See the [Squarespace OAuth documentation](https://developers.squarespace.com/commerce-apis/oauth) for more details.
