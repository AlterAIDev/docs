The SDK exposes two top-level client classes — `App` for application/operator credentials and `Agent` for workload-scoped credentials. Both are async and share the request surface defined in [`request()`](/reference/python-sdk/request).

```python
from alter_sdk import App, Agent, CallerType

# Application-side
app = App(api_key="alter_key_app_…")

# Workload-side
agent = Agent(api_key="alter_key_agent_…")

# Lifecycle
await app.close()
async with Agent(api_key="alter_key_agent_…") as a:
    ...
```

## App

The application-side client. Use for operator/admin work (provisioning agents, minting Connect sessions, managing grants) and for application-backend workloads acting under a stored `grant_id`.

```python
App(
    api_key: str,
    *,
    timeout: float = 30.0,
    caller: str | None = None,
    caller_type: CallerType | str = CallerType.SERVICE,
    user_token_getter: Callable[[], str | Awaitable[str]] | None = None,
)
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `api_key` | `str` | — | App API key (`alter_key_app_…`). Required. |
| `timeout` | `float` | `30.0` | HTTP request timeout in seconds. |
| `caller` | `str \| None` | `None` | Optional caller identifier for audit attribution. |
| `caller_type` | `CallerType \| str` | `CallerType.SERVICE` | `SERVICE` (default — backend infrastructure) or `AGENT` (shows in the Agents tab). |
| `user_token_getter` | callable | `None` | Optional sync/async callable returning a user JWT for identity-mode `request()`. |

### Properties

| Property | Type | Description |
|---|---|---|
| `actor_id` | `str \| None` | Cached actor UUID once the first authenticated call resolves. |
| `last_retry_info` | `RetryInfo \| None` | Retry metadata from the most recent `request()` call. |
| `base_url` | `str` | Backend base URL the client is pinned to. Diagnostic. |

### Methods

`App` exposes:

- [`request()`](/reference/python-sdk/request), [`proxy_request()`](/reference/python-sdk/proxy), [`boto3_client()`](/reference/python-sdk/aws)
- [`list_grants()`](/reference/python-sdk/grants), [`revoke_grant()`](/reference/python-sdk/grants), [`revoke_delegation()`](/reference/python-sdk/grants), [`create_managed_secret_grant()`](/reference/python-sdk/grants)
- [`create_connect_session()`](/reference/python-sdk/connect), [`create_managed_secret_connect_session()`](/reference/python-sdk/connect), [`connect()`](/reference/python-sdk/connect), [`poll_connect_session()`](/reference/python-sdk/connect), [`create_connect_session_for_error()`](/reference/python-sdk/connect)
- [`authenticate()`](/reference/python-sdk/connect#authenticate), `verify_user_token()`
- [`get_approval_status()`](/reference/python-sdk/approvals), [`await_approval()`](/reference/python-sdk/approvals)
- `agents` namespace (see [Agents](/reference/python-sdk/agents))
- `keys` namespace (see [Keys](/reference/python-sdk/keys))
- `scopes` namespace (see [Scopes](/reference/python-sdk/scopes))
- `with_constraints()`, `get_agent()` (below)

### with_constraints

Return a macaroon-style constrained sub-client. Every request from the returned `App` carries a permanent scope-constraints assertion that narrows the underlying key to the supplied scope set. The supplied scopes must already be implied by the key — the backend rejects broadening attempts with HTTP 400 `constraint_not_narrowing`.

```python
App.with_constraints(*, scopes: list[str]) -> App
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `scopes` | `list[str]` | — | Non-empty list of scope strings to attenuate to. |

**Raises:** `AlterValueError` (empty list, nested constraint call), `AlterSDKError` (client closed).

```python
narrowed = app.with_constraints(scopes=["grants:read"])
await narrowed.list_grants()
```

### get_agent

<Warning>
**Impersonation, NOT delegation-scoped access.** This method is for audit attribution — it stamps subsequent API calls so they appear as having been made by `agent_id`. It does NOT restrict the resulting `Agent` to that agent's delegations.

The underlying client carries the same app key (operator credentials) with `caller=<agent_id>`. Paths that bypass the delegation gate — including `request(grant_id=<user-owned uuid>)` — remain reachable. The returned `Agent` can read user OAuth tokens the named agent has no delegation row for.

Use a genuine agent key (`Agent(api_key="alter_key_agent_…")`) when delegation-scoped access is the actual security requirement. The first `request(grant_id=…)` call emits a one-time `UserWarning` to surface this footgun.
</Warning>

```python
App.get_agent(agent_id: str | UUID) -> Agent
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `agent_id` | `str \| UUID` | — | Managed-agent UUID. Must parse as a UUID. |

**Returns:** an [`Agent`](#agent) instance impersonating `agent_id`.

**Raises:** `AlterValueError` (empty / non-UUID input, App has been closed).

```python
agent = app.get_agent(agent_id="11111111-2222-3333-4444-555555555555")
me = await agent.me()
```

The parent `App`'s `user_token_getter` is intentionally NOT inherited. To bridge a user JWT through the impersonated agent, construct `Agent(api_key=…, user_token_getter=…)` directly.

### Lifecycle

```python
await app.close()

async with App(api_key="alter_key_app_…") as app:
    ...
```

`close()` shuts down the underlying HTTP clients and drains background tasks. The async context manager is the recommended pattern.

## Agent

The workload SDK client. Use for AI agents and any code that should be identity-scoped to a managed agent. The agent can reach two kinds of credentials:

1. **User-delegated OAuth grants** — a user completed Connect with `agent=<this-agent>`, creating a per-agent delegation record that authorizes the agent against the user's grant.
2. **Agent-owned managed-secret grants** — an operator provisioned a managed secret directly to the agent via the Developer Portal or `App.create_managed_secret_grant(principal={"type": "agent", …})`.

`Agent.list_grants()` returns both in one merged view.

```python
Agent(
    api_key: str,
    *,
    timeout: float = 30.0,
    caller: str | None = None,
    user_token_getter: Callable[[], str | Awaitable[str]] | None = None,
)
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `api_key` | `str` | — | Agent API key (`alter_key_agent_…`). Required. |
| `timeout` | `float` | `30.0` | HTTP request timeout in seconds. |
| `caller` | `str \| None` | `None` | Optional caller identifier for audit attribution. |
| `user_token_getter` | callable | `None` | Optional sync/async callable returning a user JWT. Used for Connect-time delegation and per-call delegation disambiguation. The JWT does NOT grant access — the agent's authority comes from the per-agent delegation record bound at Connect time. |

`Agent` pins `caller_type=AGENT` internally; it is not constructor-configurable.

### Properties

| Property | Type | Description |
|---|---|---|
| `actor_id` | `str \| None` | Cached actor UUID once the first authenticated call resolves. |
| `last_retry_info` | `RetryInfo \| None` | Retry metadata from the most recent `request()` call. |
| `base_url` | `str` | Backend base URL the client is pinned to. |
| `keys` | namespace | Scoped-key lifecycle (see [Keys](/reference/python-sdk/keys)). |
| `scopes` | namespace | Scope catalog discovery (see [Scopes](/reference/python-sdk/scopes)). |

### Methods

`Agent` exposes:

- [`me()`](/reference/python-sdk/agents#me) — self-introspection (agent-only).
- [`request()`](/reference/python-sdk/request), [`proxy_request()`](/reference/python-sdk/proxy), [`boto3_client()`](/reference/python-sdk/aws)
- [`list_grants()`](/reference/python-sdk/grants)
- [`create_connect_session()`](/reference/python-sdk/connect), [`connect()`](/reference/python-sdk/connect)
- [`revoke_delegation()`](/reference/python-sdk/grants) — self-revoke only.
- [`get_approval_status()`](/reference/python-sdk/approvals), [`await_approval()`](/reference/python-sdk/approvals)
- [`trace()`](/reference/python-sdk/agents#trace) — audit-scope context manager.
- `with_constraints()` (below)

Operator surfaces (`agents` namespace CRUD, `authenticate`, `verify_user_token`, `revoke_grant`, `create_managed_secret_grant`) are intentionally absent — calling them with an agent key would raise typed errors deep in the request path.

### with_constraints

Identical semantics to [`App.with_constraints()`](#with-constraints) — returns a constrained sub-`Agent` whose every request carries the scope-constraints assertion.

```python
Agent.with_constraints(*, scopes: list[str]) -> Agent
```

### trace

Async context manager that scopes audit identity for every nested `request()` call. See [`Agent.trace()`](/reference/python-sdk/agents#trace) on the Agents page for the full surface.

### Lifecycle

```python
await agent.close()

async with Agent(api_key="alter_key_agent_…") as agent:
    ...
```

## is_valid_key

Validate the syntactic shape of an Alter API key without a network round-trip. Re-derives the CRC checksum from the key body and compares — catches single-character transcription errors.

```python
from alter_sdk import is_valid_key

is_valid_key(plain_key: object) -> bool
```

| Parameter | Type | Default | Description |
|---|---|---|---|
| `plain_key` | `object` | — | Candidate key string. Non-strings and empty strings return `False`. |

Returns `True` only for syntactically-valid keys (legacy `alter_key_*` prefix-only, or scoped `alter_<type>_<random>_<checksum>` with a matching CRC). The check is a usability gate — a forged checksum still cannot pass backend verification.

```python
from alter_sdk import App, is_valid_key

raw = input("Paste API key: ").strip()
if not is_valid_key(raw):
    raise SystemExit("Invalid key shape")
app = App(api_key=raw)
```
