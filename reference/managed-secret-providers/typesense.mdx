## Overview

Use Typesense credentials to make authenticated API calls through Alter Vault without exposing API keys in code.

| Property | Value |
|----------|-------|
| **Provider ID** | `typesense` |
| **Category** | Search |
| **Credential Type** | API Key |

## Step 1: Get Credentials

<Steps>
  <Step title="Log in to Typesense Cloud">
    Log in to [Typesense Cloud](https://cloud.typesense.org/) and open the cluster.
  </Step>
  <Step title="Generate a key">
    Open **API Keys** and generate a key — use a scoped search-only key when the integration only reads.
  </Step>
  <Step title="Copy the key">
    Copy the key — it is only shown once.
  </Step>
</Steps>

<Warning>
  Never share credentials publicly. Store them securely and only enter them in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterauth.com](https://portal.alterauth.com) and navigate to the application.
  </Step>

  <Step title="Add Typesense">
    Go to **Managed Secrets** > **Add Provider** > **Typesense**.
  </Step>

  <Step title="Enter credentials">
    Paste your API Key into the credential field.
  </Step>

  <Step title="Save">
    Click **Save**. You'll receive a `grant_id` to use with the SDK.
  </Step>
</Steps>

## Using in Code

```python
response = await alter_app.request(
    HttpMethod.GET,
    "https://YOUR-CLUSTER.a1.typesense.net/collections/products/documents/search",
    grant_id="YOUR_GRANT_ID",
    query_params={"q": "running shoes", "query_by": "name,description"},
)
```

## Notes

- The key is injected via the `X-TYPESENSE-API-KEY` header.
