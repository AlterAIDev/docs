## Overview

Use ClickHouse credentials to make authenticated API calls through Alter Vault without exposing API keys in code.

| Property | Value |
|----------|-------|
| **Provider ID** | `clickhouse` |
| **Category** | Database |
| **Credential Type** | Base64 Credentials |

## Step 1: Get Credentials

<Steps>
  <Step title="Log in to ClickHouse Cloud">
    Log in to the [ClickHouse Cloud Console](https://console.clickhouse.cloud/) and open the service.
  </Step>
  <Step title="Note the connection details">
    Open **Connect** and note the HTTPS endpoint, username, and password (or create a dedicated database user).
  </Step>
  <Step title="Encode the credentials">
    Base64-encode the credentials: `echo -n 'username:password' | base64` and paste the result into the credential field.
  </Step>
</Steps>

<Warning>
  Never share credentials publicly. Store them securely and only enter them in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterauth.com](https://portal.alterauth.com) and navigate to the application.
  </Step>

  <Step title="Add ClickHouse">
    Go to **Managed Secrets** > **Add Provider** > **ClickHouse**.
  </Step>

  <Step title="Enter credentials">
    Paste your Base64 Credentials into the credential field.
  </Step>

  <Step title="Save">
    Click **Save**. You'll receive a `grant_id` to use with the SDK.
  </Step>
</Steps>

## Using in Code

```python
response = await alter_app.request(
    HttpMethod.POST,
    "https://YOUR-SERVICE.clickhouse.cloud:8443",
    grant_id="YOUR_GRANT_ID",
    query_params={"query": "SELECT event, count() FROM events GROUP BY event LIMIT 5"},
)
```

## Notes

- The HTTPS interface accepts SQL in the request body and listens on port 8443 on ClickHouse Cloud.
