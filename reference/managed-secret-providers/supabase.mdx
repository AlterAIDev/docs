## Overview

Use Supabase credentials to make authenticated API calls through Alter Vault without exposing API keys in code.

| Property | Value |
|----------|-------|
| **Provider ID** | `supabase` |
| **Category** | Developer Tools |
| **Credential Type** | Bearer Token |

## Step 1: Get Credentials

<Steps>
  <Step title="Log in to the Supabase Dashboard">
    Log in to the Supabase Dashboard at [supabase.com/dashboard](https://supabase.com/dashboard/).
  </Step>
  <Step title="Select your project">
    Select your project.
  </Step>
  <Step title="Go to Settings → API">
    Go to Settings → API.
  </Step>
  <Step title="Copy your API key">
    Copy the 'anon' key (safe for client-side) or 'service_role' key (server-side only, bypasses RLS).
  </Step>
  <Step title="Copy the Project URL">
    Copy the Project URL (e.g. your-project.supabase.co) for the Project URL field.
  </Step>
</Steps>

<Warning>
  Never share credentials publicly. Store them securely and only enter them in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterauth.com](https://portal.alterauth.com) and navigate to the application.
  </Step>

  <Step title="Add Supabase">
    Go to **Managed Secrets** > **Add Provider** > **Supabase**.
  </Step>

  <Step title="Enter credentials">
    Paste your API Key and Project URL into their respective fields.
  </Step>

  <Step title="Save">
    Click **Save**. You'll receive a `grant_id` to use with the SDK.
  </Step>
</Steps>

## Using in Code

```python
response = await alter_app.request(
    HttpMethod.GET,
    "https://your-project.supabase.co/rest/v1/your_table",
    grant_id="YOUR_GRANT_ID",
)
```
