## Overview

Use Neon credentials to make authenticated API calls through Alter Vault without exposing API keys in code.

| Property | Value |
|----------|-------|
| **Provider ID** | `neon` |
| **Category** | Database |
| **Credential Type** | API Key |

## Step 1: Get Credentials

<Steps>
  <Step title="Log in to the Neon Console">
    Log in to the [Neon Console](https://console.neon.tech/).
  </Step>
  <Step title="Open API keys">
    Go to [Account settings > API keys](https://console.neon.tech/app/settings/api-keys) — or Organization settings for an org/project-scoped key.
  </Step>
  <Step title="Create a key">
    Click **Create new API key** and name it.
  </Step>
  <Step title="Copy the key">
    Copy the key (starts with `napi_`) — it is only shown once.
  </Step>
</Steps>

<Warning>
  Never share credentials publicly. Store them securely and only enter them in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterauth.com](https://portal.alterauth.com) and navigate to the application.
  </Step>

  <Step title="Add Neon">
    Go to **Managed Secrets** > **Add Provider** > **Neon**.
  </Step>

  <Step title="Enter credentials">
    Paste your API Key into the credential field.
  </Step>

  <Step title="Save">
    Click **Save**. You'll receive a `grant_id` to use with the SDK.
  </Step>
</Steps>

## Using in Code

```python
response = await alter_app.request(
    HttpMethod.GET,
    "https://console.neon.tech/api/v2/projects",
    grant_id="YOUR_GRANT_ID",
)
```

## Notes

- The Neon API manages projects, branches, computes, databases, and roles — ideal for provisioning an isolated Postgres branch per agent or per session.
- Running SQL uses the Postgres protocol or Neon's serverless driver against the branch's connection string, not this API key.
