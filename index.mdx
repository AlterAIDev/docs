Alter Vault stores OAuth tokens and API credentials in an encrypted vault, enforces identity-aware authorization on every retrieval, and logs every access with full actor context. Application code becomes a single call: `app.request(method, url, grant_id=...)`.

## Why Alter Vault

- **Zero token exposure.** Credentials never leave the vault. Application code only ever holds an API key.
- **Identity-aware authorization.** Every credential access is checked against both the end-user identity and the workload identity. One [policy layer](/reference/policies) governs which callers reach which credentials.
- **Dual-identity audit logging.** Every access is logged with both the end-user identity and the caller identity. Answer "what was Alice's data used for?" and "what did Agent X access?"
- **Fail-closed by design.** When authorization cannot be evaluated, credential access is denied.
- **Drop-in replacement.** `app.request()` is the application's HTTP client — same surface as `httpx` or `fetch`, with the credential injected automatically.

## Quick example

This is the call written in [First 5 Minutes](/first-5-minutes). It posts a Slack message; the token never touches application code.

<CodeGroup>
```python Python
from alter_sdk import App, HttpMethod

app = App(api_key=ALTER_API_KEY)
response = await app.request(
    HttpMethod.POST,
    "https://slack.com/api/chat.postMessage",
    grant_id=SLACK_GRANT_ID,
    json={"channel": "#general", "text": "Hello from Alter Vault."},
)
```

```typescript TypeScript
import { App, HttpMethod } from "@alter-ai/alter-sdk";

const app = new App({ apiKey: ALTER_API_KEY });
const response = await app.request(
  HttpMethod.POST,
  "https://slack.com/api/chat.postMessage",
  { grantId: SLACK_GRANT_ID, json: { channel: "#general", text: "Hello from Alter Vault." } },
);
```
</CodeGroup>
