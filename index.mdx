## Why Alter Vault?

### Token Vault for Downstream Providers
OAuth tokens and API keys are encrypted and stored in a dedicated vault, never in the application database. Token refresh, lifecycle management, and credential injection happen automatically through `vault.request()`.

### Identity-Aware Authorization
Every credential access is checked against two identities: the end user whose credentials are being used, and the agent requesting them. One [policy layer](/reference/policies) governs which actors get access to which credentials, down to the provider and action level.

### Dual-Identity Audit Logging
Every access is logged with both the end-user identity and the actor identity: which agent, which tool, which action. Answer "what was Alice's data used for?" and "what did Agent X access across all users?"

### Security and Compliance
Fail-closed by design: if authorization cannot be enforced, credential access is denied. Every access is logged for SOC 2, GDPR, and HIPAA compliance.

## How It Works

Alter Vault manages two types of credentials -**OAuth grants** and **managed secrets** -both accessed through the same `vault.request()` method.

`vault.request()` **acts as the HTTP client**. Instead of calling `fetch` or `httpx` directly and constructing auth headers manually, call `vault.request()` with a `grant_id` and the target URL. The SDK makes the HTTP request with the correct credential automatically injected. Application code never sees, stores, or handles tokens or secrets, eliminating the risk of accidental leakage through logs, error messages, or insecure storage.

| | OAuth Connections | Managed Secrets |
|---|---|---|
| **Use case** | Third-party services (Google, Slack, GitHub) | Internal APIs and services (API keys, service tokens) |
| **Who provides credentials** | **End user** authorizes via OAuth | **Developer** stores via portal |
| **Where `grant_id` comes from** | `onSuccess` callback after user completes OAuth | Developer Portal when storing a secret |

<Steps>
  <Step title="Get a grant_id">
    **OAuth**: The end user authorizes via Alter Connect UI → `onSuccess` returns a `grant_id`. **Managed Secrets**: Store credentials in the Developer Portal → portal returns a `grant_id`.
  </Step>

  <Step title="We Encrypt & Store">
    All credentials are encrypted (AES-256-GCM) and stored in a secure vault, never in the application database
  </Step>

  <Step title="Call vault.request()">
    `vault.request()` acts as the HTTP client, making the API call and injecting the right credential automatically. The token never touches application code.
  </Step>

  <Step title="Alter Handles the Rest">
    Token refresh, policy enforcement, audit logging, all automatic
  </Step>
</Steps>

## Our SDKs

<Info>
**SDKs and tools:**
- **Python / TypeScript SDK (`alter-sdk` / `@alter-ai/alter-sdk`)** - Backend SDK for creating OAuth sessions and making authenticated API calls via `vault.request()`. Works with both OAuth grants and managed secrets.
- **Alter Connect UI** - Hosted page where users complete OAuth. The backend creates a session and gets a `connect_url` that works in any browser. No frontend SDK required.
- **Connect SDK (`@alter-ai/connect`)** - **Optional** frontend SDK that opens the Alter Connect UI in a popup with `onSuccess`/`onError` callbacks. Useful for SPAs, but not required.
</Info>

## Quick Example

<CodeGroup>

```python Python (Headless)
import asyncio
from alter_sdk import AlterVault, ActorType, HttpMethod

async def main():
    vault = AlterVault(
        api_key="alter_key_...",
        actor_type=ActorType.AI_AGENT,
        actor_identifier="my-agent",
    )

    # Open browser → user authorizes → SDK returns connections
    results = await vault.connect(
        providers=["google"],
    )

    # Make API calls with the grant
    response = await vault.request(
        results[0].grant_id,
        HttpMethod.GET,
        "https://www.googleapis.com/calendar/v3/calendars/primary/events",
        query_params={"maxResults": "10"},
    )
    events = response.json()
    print(f"Found {len(events.get('items', []))} events")

    await vault.close()

asyncio.run(main())
```

```javascript JavaScript (Web App)
import AlterConnect from '@alter-ai/connect';

// Create Connect instance
const alterConnect = AlterConnect.create();

// Open OAuth flow
async function connectProvider() {
    // Call YOUR backend to create a session
    const { session_token } = await fetch('/api/connect/session')
        .then(r => r.json());

    await alterConnect.open({
        token: session_token,
        onSuccess: (connections) => {
            // User completed OAuth! Store each grant_id in your DB.
            for (const conn of connections) {
                console.log('Connected!', conn.grant_id, conn.provider);
            }
        }
    });
}
```

```python Backend (Session Creation)
from alter_sdk import AlterVault, ActorType

vault = AlterVault(
    api_key="alter_key_...",
    actor_type=ActorType.BACKEND_SERVICE,
    actor_identifier="my-app",
)

# Create session for frontend
session = await vault.create_connect_session()
session_token = session.session_token
```
