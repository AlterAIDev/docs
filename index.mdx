## Identity-aware access control for AI agents and applications

Alter Vault securely stores OAuth tokens and API credentials on behalf of your users, and enforces fine-grained permissions so every retrieval, tool call, and action respects what each user and agent is allowed to access, without rebuilding auth in every workflow.

## Why Alter Vault?

### Token Storage You Don't Have to Build
OAuth tokens and API keys are encrypted (AES-256-GCM) and stored in a dedicated vault. Your database never holds credentials. Token refresh happens automatically. You call `vault.request()` and the right credential is injected into the outgoing HTTP request.

### Policies That Block, Not Filter
Every `vault.request()` call is checked against policy before the credential is released. If the policy says no, the request is blocked. Policies take both identities into account: which agent is calling and which end user it is acting for. For example, you can allow an agent to read a user's Google Calendar but deny access to their Gmail. Scope rules further by HTTP method, URL pattern, and connection.

### End-User Identity Built In
Alter Vault resolves the end user behind each agent request via JWT from Auth0, Clerk, Supabase, or any OIDC provider. Both identities (which agent and which user) are carried through every policy check and audit log, so access control is never just app-level.

### One Method for Retrieval and Actions
`vault.request()` handles reads and writes. Whether your agent fetches Google Calendar events, posts a Slack message, or creates a GitHub issue, the same method injects credentials, enforces policy, and logs the call.

### Full Audit Trail with Actor Context
Every token access, policy decision, and API call is logged with agent ID, run ID, tool call ID, and end-user identity. Filter and export logs for SOC 2, GDPR, and HIPAA compliance.

## How It Works

Alter Vault manages two types of credentials -**OAuth connections** and **managed secrets** -both accessed through the same `vault.request()` method.

`vault.request()` **is your HTTP client**. Instead of calling `fetch` or `httpx` directly (and constructing auth headers yourself), you call `vault.request()` with a `connection_id` and the target URL. The SDK makes the HTTP request on your behalf with the correct credential automatically injected -your code never sees, stores, or handles any tokens or secrets. This eliminates the risk of accidental token leakage through logs, error messages, or insecure storage.

| | OAuth Connections | Managed Secrets |
|---|---|---|
| **Use case** | Third-party services (Google, Slack, GitHub) | Your own APIs (internal services, API keys) |
| **Who provides credentials** | **End user** authorizes via OAuth | **Developer** stores via portal |
| **Where `connection_id` comes from** | `onSuccess` callback after user completes OAuth | Developer Portal when you store a secret |

<Steps>
  <Step title="Get a connection_id">
    **OAuth**: Your end user authorizes via Alter Connect UI → `onSuccess` returns a `connection_id`. **Managed Secrets**: You store credentials in the Developer Portal → portal returns a `connection_id`.
  </Step>

  <Step title="We Encrypt & Store">
    All credentials are encrypted (AES-256-GCM) and stored in our secure vault -never in your database
  </Step>

  <Step title="You Call vault.request()">
    `vault.request()` acts as your HTTP client -it makes the API call and injects the right credential automatically. You get back the API response. The token never touches your code.
  </Step>

  <Step title="We Handle the Rest">
    Token refresh, policy enforcement, audit logging -all automatic
  </Step>
</Steps>

## Our SDKs

<Info>
**SDKs and tools:**
- **Python / TypeScript SDK (`alter-sdk` / `@alter-ai/alter-sdk`)** - Backend SDK for creating OAuth sessions and making authenticated API calls via `vault.request()`. Works with both OAuth connections and managed secrets.
- **Alter Connect UI** - Hosted page where users complete OAuth. Your backend creates a session and gets a `connect_url` that works in any browser. No frontend SDK required.
- **Connect SDK (`@alter-ai/connect`)** - **Optional** frontend SDK that opens the Alter Connect UI in a popup with `onSuccess`/`onError` callbacks. Useful for SPAs, but not required.
</Info>

## Quick Example

<CodeGroup>

```python Python (Headless)
import asyncio
from alter_sdk import AlterVault, ActorType, HttpMethod

async def main():
    vault = AlterVault(
        api_key="alter_key_...",
        actor_type=ActorType.AI_AGENT,
        actor_identifier="my-agent",
    )

    # Open browser → user authorizes → SDK returns connections
    results = await vault.connect(
        providers=["google"],
    )

    # Make API calls with the connection
    response = await vault.request(
        results[0].connection_id,
        HttpMethod.GET,
        "https://www.googleapis.com/calendar/v3/calendars/primary/events",
        query_params={"maxResults": "10"},
    )
    events = response.json()
    print(f"Found {len(events.get('items', []))} events")

    await vault.close()

asyncio.run(main())
```

```javascript JavaScript (Web App)
import AlterConnect from '@alter-ai/connect';

// Create Connect instance
const alterConnect = AlterConnect.create();

// Open OAuth flow
async function connectProvider() {
    // Call YOUR backend to create a session
    const { session_token } = await fetch('/api/connect/session')
        .then(r => r.json());

    await alterConnect.open({
        token: session_token,
        onSuccess: (connections) => {
            // User completed OAuth! Store each connection_id in your DB.
            for (const conn of connections) {
                console.log('Connected!', conn.connection_id, conn.provider);
            }
        }
    });
}
```

```python Backend (Session Creation)
from alter_sdk import AlterVault, ActorType

vault = AlterVault(
    api_key="alter_key_...",
    actor_type=ActorType.BACKEND_SERVICE,
    actor_identifier="my-app",
)

# Create session for frontend
session = await vault.create_connect_session()
session_token = session.session_token
```
