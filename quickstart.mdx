## Overview

This guide will help you integrate Alter Vault into your application. By the end, you'll be able to:
- Connect OAuth accounts and make authenticated API calls
- Use the Python SDK with automatic token injection
- Never handle OAuth tokens directly

## Two Types of Credentials

Alter Vault manages two types of credentials. Both use the same `vault.request()` method -the only difference is **where the `connection_id` comes from**.

| | OAuth Connections | Managed Secrets |
|---|---|---|
| **What** | Third-party services (Google, Slack, GitHub) | Your own APIs (internal services, partner APIs) |
| **Who provides credentials** | **End user** authorizes via OAuth flow | **Developer** stores via Developer Portal |
| **Where `connection_id` comes from** | Returned in `onSuccess` callback when user completes Alter Connect | Returned by Developer Portal when you store a secret |
| **Setup** | Alter Connect UI (frontend) or `vault.connect()` (headless) | Developer Portal → Managed Secrets → Store Secret |
| **Token refresh** | Automatic (OAuth refresh flow) | Manual (re-store when rotated) |

<Info>
**The key distinction**: For OAuth, your **end user** triggers the connection by logging into their account (Google, Slack, etc.), and the `connection_id` is returned to you. For Managed Secrets, **you** (the developer) store the credential in the portal and get the `connection_id` back. Once you have a `connection_id`, usage is identical: `vault.request(connection_id, ...)`.
</Info>

## Two Integration Paths

| Path | Best for | What you need |
|------|----------|---------------|
| **Quick path** (below) | CLI tools, scripts, Jupyter notebooks, prototyping | Python only |
| **Full web app** (Step 4+) | Production web apps with a frontend | Python + JavaScript |

## Quick Path: Connect from Python

The fastest way to get started. Everything happens in Python -- the SDK opens a browser for the user to authorize, then returns the connection.

```python
import asyncio
from alter_sdk import AlterVault, ActorType, HttpMethod

async def main():
    vault = AlterVault(
        api_key="alter_key_...",
        actor_type=ActorType.AI_AGENT,
        actor_identifier="my-agent",
    )

    # Open browser → user authorizes → SDK returns connections
    results = await vault.connect(
        providers=["google"],
    )
    print(f"Connected: {results[0].connection_id} ({results[0].provider_id})")

    # Make API calls with the connection
    response = await vault.request(
        results[0].connection_id,
        HttpMethod.GET,
        "https://www.googleapis.com/calendar/v3/calendars/primary/events",
        query_params={"maxResults": "10"},
    )
    events = response.json()
    for event in events.get("items", []):
        print(f"Event: {event['summary']}")

    await vault.close()

asyncio.run(main())
```

That's it. No frontend code, no session tokens, no callbacks. The SDK handles the entire OAuth flow.

<Note>
  `connect()` opens the user's default browser. Set `open_browser=False` to print the URL instead (useful for remote servers).
</Note>

Skip to [Step 6](#step-6-use-the-python-sdk) for more usage examples, or continue below for the full web app integration.

## Full Web App Integration

For production web apps, use the session-based flow. Your backend creates a session and gets back a `connect_url` - a hosted page where the user completes OAuth.

### Choosing Your OAuth Trigger Method

| Method | When to use | How it works |
|--------|------------|--------------|
| **Headless** (`vault.connect()`) | CLI tools, scripts, Jupyter notebooks, backend agents. No frontend at all. | One SDK call does everything — creates a session, opens the user's default browser, and polls until OAuth completes. Everything happens in Python/TypeScript. |
| **Redirect** | Server-rendered apps (Django, Rails, etc.), email links, or any app that can redirect a browser. No JavaScript SDK needed. | Your backend calls `create_connect_session()`, gets a `connect_url`, and redirects the user to it. After OAuth, the user is redirected back to your `return_url` with the results. |
| **Popup** (`@alter-ai/connect` SDK) | Single-page apps (React, Vue, etc.) where you want a popup overlay with real-time `onSuccess`/`onError` callbacks without leaving the page. | Your backend calls `create_connect_session()` and passes the `session_token` to the frontend SDK, which opens the hosted page in a popup. |

<Info>
**The frontend JavaScript SDK (`@alter-ai/connect`) is optional.** The `connect_url` returned by `create_connect_session()` is a standard HTTPS URL that works in any browser. The frontend SDK just wraps it in a popup with callbacks for a smoother SPA experience.
</Info>

### Key Components

1. **Python / TypeScript SDK** - Backend SDK that creates sessions and makes authenticated API calls via `vault.request()`
2. **Alter Connect UI** - Hosted page (at the `connect_url`) that handles provider selection and the entire OAuth flow
3. **Connect SDK (`@alter-ai/connect`)** - **Optional** frontend SDK that opens the hosted page in a popup with callbacks

## Step 4: Create a Session

Your backend calls `create_connect_session()` which returns:
- **`connect_url`** - a hosted page where the user completes OAuth (use this for the redirect flow)
- **`session_token`** - a short-lived token to pass to the frontend SDK (use this for the popup flow)

<CodeGroup>

```python Flask (Python SDK)
import asyncio
from flask import Flask, request, jsonify, redirect
from alter_sdk import AlterVault, ActorType

app = Flask(__name__)

vault = AlterVault(
    api_key="alter_key_...",
    actor_type=ActorType.BACKEND_SERVICE,
    actor_identifier="my-flask-app",
)

@app.route('/api/connect/session', methods=['POST'])
def create_oauth_session():
    user = request.user  # Your auth

    async def _create():
        session = await vault.create_connect_session(
            allowed_providers=["google", "slack", "github"],
        )
        return session

    session = asyncio.run(_create())

    # Return both - your frontend decides which to use
    return jsonify({
        "session_token": session.session_token,  # For popup flow
        "connect_url": session.connect_url,       # For redirect flow
    })
```

```javascript Express (TypeScript SDK)
const express = require('express');
const { AlterVault, ActorType } = require('@alter-ai/alter-sdk');

const app = express();

const vault = new AlterVault({
  apiKey: 'alter_key_...',
  actorType: ActorType.BACKEND_SERVICE,
  actorIdentifier: 'my-express-app',
});

app.post('/api/connect/session', async (req, res) => {
    const user = req.user; // Your auth

    const session = await vault.createConnectSession({
        allowedProviders: ['google', 'slack', 'github'],
    });

    // Return both - your frontend decides which to use
    res.json({
        session_token: session.sessionToken,  // For popup flow
        connect_url: session.connectUrl,       // For redirect flow
    });
});
```

</CodeGroup>

## Step 5: Trigger the OAuth Flow

Choose the approach that fits your app:

### Option A: Redirect Flow (no frontend SDK needed)

Redirect the user's browser to the `connect_url`. After OAuth completes, the user is redirected back to your app. This works with any backend framework - no JavaScript SDK required.

<CodeGroup>

```python Flask - Server-side redirect
@app.route('/connect/<provider>')
def connect_provider(provider):
    user = request.user  # Your auth

    async def _create():
        session = await vault.create_connect_session(
            allowed_providers=[provider],
            return_url="https://myapp.com/connect/callback",
        )
        return session

    session = asyncio.run(_create())
    return redirect(session.connect_url)  # User goes to hosted OAuth page

@app.route('/connect/callback')
def connect_callback():
    # User returns here after completing OAuth
    # The connection_id is available via vault.list_connections()
    return "Connected! You can close this tab."
```

```javascript Frontend - window.location redirect
async function connectProvider(provider) {
    const response = await fetch('/api/connect/session', {
        method: 'POST',
        headers: { 'Authorization': `Bearer ${authToken}` }
    });
    const { connect_url } = await response.json();

    // Navigate to the hosted OAuth page - no SDK needed
    window.location.href = connect_url;
}
```

</CodeGroup>

### Option B: Popup Flow (frontend SDK)

Use the `@alter-ai/connect` SDK to open the OAuth page in a popup with real-time callbacks. The user stays on your page.

<CodeGroup>

```javascript React
import AlterConnect from '@alter-ai/connect';

        await alterConnect.open({
            token: session_token,
            onSuccess: (connections) => {
                // User completed OAuth! Store each connection_id in your database.
                for (const conn of connections) {
                    console.log('Connected!', conn.connection_id, conn.provider);
                }
                loadIntegrations();
            },
            onError: (error) => {
                console.error('Connection failed', error);
            }
        });
    };

    return (
        <div>
            <button onClick={() => handleConnect('google')}>
                Connect Google
            </button>
            <button onClick={() => handleConnect('slack')}>
                Connect Slack
            </button>
        </div>
    );
}
```

## Step 6: Use the Python SDK

Now use the SDK to make API calls with automatic token injection:

### Basic Usage

```python
import asyncio
from alter_sdk import AlterVault, ActorType, HttpMethod

async def fetch_calendar_events(user_id: str):
    vault = AlterVault(
        api_key="alter_key_...",
        actor_type=ActorType.AI_AGENT,
        actor_identifier="my-agent",
    )

    # Your app stores connection_id when the user completes Alter Connect (Step 5)
    connection_id = db.get_connection(user_id=user_id, provider="google")

    try:
        # Make API request -token is injected automatically
        response = await vault.request(
            connection_id,
            HttpMethod.GET,
            "https://www.googleapis.com/calendar/v3/calendars/primary/events",
            query_params={
                "maxResults": "10",
                "singleEvents": "true",
                "orderBy": "startTime",
            },
            reason="Fetching calendar events",
        )

        events = response.json()
        for event in events.get("items", []):
            start = event["start"].get("dateTime", event["start"].get("date"))
            print(f"{start}: {event.get('summary', 'No title')}")

        return events

    finally:
        await vault.close()

asyncio.run(fetch_calendar_events("alice"))
```

### Error Handling

```python
from alter_sdk.exceptions import (
    ReAuthRequiredError,
    PolicyViolationError,
    ConnectionNotFoundError,
    NetworkError,
    ScopeReauthRequiredError,
    ProviderAPIError,
)

try:
    response = await vault.request(
        connection_id, HttpMethod.GET,
        "https://www.googleapis.com/calendar/v3/calendars/primary/events",
    )
except ReAuthRequiredError:
    print("User must re-authorize - redirect to Alter Connect")
except PolicyViolationError as e:
    print(f"Access denied by policy: {e.message} (rule: {e.policy_error})")
except ConnectionNotFoundError:
    print("User hasn't connected Google yet - redirect to Alter Connect")
except NetworkError as e:
    print(f"Network issue: {e.message}")
except ScopeReauthRequiredError as e:
    print(f"Scopes outdated for {e.provider_id} - user must re-authorize")
except ProviderAPIError as e:
    print(f"Google API error {e.status_code}: {e.response_body}")
```

## Complete Example: Flask + Google Calendar

Here's a full example showing a Flask API that fetches calendar events:

```python
import asyncio
from flask import Flask, jsonify, request
from alter_sdk import AlterVault, ActorType, HttpMethod
from alter_sdk.exceptions import ConnectionNotFoundError

app = Flask(__name__)
vault = AlterVault(
    api_key="alter_key_...",
    actor_type=ActorType.BACKEND_SERVICE,
    actor_identifier="my-flask-app",
)

@app.route('/api/users/<user_id>/events')
def get_user_events(user_id):
    """Fetch user's calendar events using Alter SDK."""
    # Your app stores connection_id when the user completes Alter Connect (Step 5)
    connection_id = db.get_connection(user_id=user_id, provider="google")

    async def fetch_events():
        try:
            response = await vault.request(
                connection_id,
                HttpMethod.GET,
                "https://www.googleapis.com/calendar/v3/calendars/primary/events",
                query_params={
                    "maxResults": "10",
                    "singleEvents": "true",
                    "orderBy": "startTime",
                },
                reason="User viewing calendar events",
            )
            return response.json()
        except ConnectionNotFoundError:
            return None

    result = asyncio.run(fetch_events())

    if result is None:
        return jsonify({"error": "User hasn't connected Google account"}), 404

    return jsonify(result)

if __name__ == '__main__':
    app.run(debug=True)
```

## Alternative: Using Managed Secrets

If you're connecting to **your own APIs** (internal services, partner APIs, SaaS platforms with API keys), you don't need the OAuth flow. Use Managed Secrets instead.

<Info>
**How is this different from OAuth?** With OAuth, the `connection_id` comes from your **end user** completing an OAuth login. With Managed Secrets, the `connection_id` comes from **you** (the developer) storing a credential in the Developer Portal. No end-user action is needed.
</Info>

### Step 1: Store Your Secret in the Developer Portal

1. Go to the Developer Portal → your app → **Managed Secrets**
2. Click **Add Provider** (e.g., "Loyalty API")
3. Choose credential type (Bearer Token, API Key, Basic Auth, or AWS SigV4)
4. Click **Store Secret** and enter the credential value
5. **Copy the `connection_id` returned** -this is what you'll use in your code

### Step 2: Use vault.request()

Use the exact same `vault.request()` method -just pass the managed secret's `connection_id`:

<CodeGroup>

```python Python
from alter_sdk import AlterVault, ActorType, HttpMethod

async with AlterVault(
    api_key="alter_key_...",
    actor_type=ActorType.AI_AGENT,
    actor_identifier="my-agent",
) as vault:
    # Call your internal API -secret injected automatically
    response = await vault.request(
        "MANAGED_SECRET_CONNECTION_ID",  # from Developer Portal
        HttpMethod.GET,
        "https://api.internal.com/v1/loyalty/points",
        query_params={"user_id": "alice"},
        reason="Checking loyalty points",
    )
    points = response.json()
```

```typescript TypeScript
import { AlterVault, ActorType, HttpMethod } from "@alter-ai/alter-sdk";

const vault = new AlterVault({
  apiKey: "alter_key_...",
  actorType: ActorType.AI_AGENT,
  actorIdentifier: "my-agent",
});

const response = await vault.request(
  "MANAGED_SECRET_CONNECTION_ID",  // from Developer Portal
  HttpMethod.GET,
  "https://api.internal.com/v1/loyalty/points",
  {
    queryParams: { user_id: "alice" },
    reason: "Checking loyalty points",
  }
);
const points = await response.json();
await vault.close();
```

## Key Concepts

### Zero Token Exposure
`vault.request()` **is your HTTP client** -it replaces direct `fetch` or `httpx` calls. Instead of constructing auth headers yourself, you pass a `connection_id` and the target URL. The SDK:
- Makes the HTTP request on your behalf
- Retrieves the credential internally and injects it as the correct auth header
- Returns the API response directly to you
- Refreshes OAuth tokens transparently before they expire

Your code never sees, stores, or handles any tokens or secrets -eliminating the risk of accidental leakage through logs, error messages, or insecure storage.

### Where Does `connection_id` Come From?

Every credential in Alter Vault has a unique `connection_id` (UUID). Where it originates depends on the credential type:

- **OAuth connections**: The `connection_id` is returned in the `onSuccess` callback when your **end user** completes the OAuth flow via Alter Connect. Store this in your database, mapped to your users.
- **Managed secrets**: The `connection_id` is returned by the **Developer Portal** when you store a credential. Hard-code it in your config or environment variables.

```python
# OAuth: connection_id stored in your DB after user completed Alter Connect
connection_id = db.get_connection(user_id=user_id, provider="google")
await vault.request(connection_id, HttpMethod.GET, url)

# Managed secret: connection_id from Developer Portal, stored in your config
await vault.request(os.environ["LOYALTY_API_CONNECTION_ID"], HttpMethod.GET, url)
```

### Multiple Connections (Multi-Account)

A user can connect **multiple accounts** to the same provider (e.g., personal Gmail + work Gmail). Each connection gets its own `connection_id`.

**Store the account identifier** alongside the `connection_id` when the user completes Alter Connect:

```javascript
// In your Alter Connect onSuccess callback (Step 5)
onSuccess: async (connections) => {
    for (const conn of connections) {
        await saveToDatabase({
            user_id: currentUser.id,
            connection_id: conn.connection_id,
            provider: conn.provider,
            account: conn.account_identifier  // e.g., "alice@gmail.com"
        });
    }
}
```

**Look up connections** when making API calls:

```python
# Simple case: one connection per user per provider
connection_id = db.get_connection(user_id=user_id, provider="google")

# Multi-account: user picks which account to use
connections = db.get_connections(user_id=user_id, provider="google")
# Returns: [("aaa-111", "alice@gmail.com"), ("bbb-222", "alice@work.com")]
```

**List connections via the SDK** to see what a user has connected:

```python
result = await vault.list_connections(provider_id="google")
for conn in result.connections:
    print(f"{conn.id}: {conn.account_display_name}")
```

## Best Practices

### Security
- Keep API keys server-side only
- Use session tokens (short-lived) for frontend
- Set IP restrictions in production

### Connection Management
- Store the `connection_id` from Alter Connect in your database
- Map connection IDs to your users for easy lookup
- Use `listConnections()` to retrieve connection IDs programmatically

### Error Handling
- Always handle `PolicyViolationError` and `ConnectionNotFoundError` explicitly
- Don't catch all exceptions with a bare `except Exception`
- Use retry logic for `NetworkError` (with exponential backoff)

## Troubleshooting

### Connection Not Found
If you get `ConnectionNotFoundError`, ensure:
- User has completed OAuth flow via Alter Connect
- The `connection_id` is correct and belongs to your app
- Connection hasn't been revoked

### Token Expired
The SDK automatically refreshes tokens. If refresh fails:
- User may need to re-authenticate
- Check if refresh token has expired
- Verify OAuth app credentials are still valid

### Policy Violation
If you get `PolicyViolationError`:
- Check the error message for the specific rule that failed (time-based access or IP allowlist)
- Review the policy configuration in **App Details -> Policies** in the Developer Portal
- If using IP allowlist, confirm your server's IP is included
- If using time restrictions, check if you're within business hours / weekdays

### Provider Not Configured
If you get `BackendError` with "not configured":
- Add the provider in the Alter Vault dashboard
- Enter your OAuth Client ID and Client Secret
