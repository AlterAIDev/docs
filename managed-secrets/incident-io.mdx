## Overview

Use Incident.io credentials to make authenticated API calls through Alter Vault without exposing your API keys in code.

| Property | Value |
|----------|-------|
| **Provider ID** | `incident-io` |
| **Category** | Monitoring & Observability |
| **Credential Type** | Bearer Token |

## Step 1: Get Your Credentials

<Steps>
  <Step title="Log in to incident.io">
    Log in to incident.io at [app.incident.io](https://app.incident.io/).
  </Step>
  <Step title="Go to Settings → API Keys">
    Go to Settings → API Keys at [app.incident.io/settings/api-keys](https://app.incident.io/settings/api-keys).
  </Step>
  <Step title="Create an API Key">
    Click 'Create API Key' and configure permissions.
  </Step>
  <Step title="Copy your API key">
    Copy the generated API key.
  </Step>
</Steps>

<Warning>
  Never share your credentials publicly. Store them securely and only enter them in the Alter Vault Developer Portal.
</Warning>

## Step 2: Add to Alter Vault

<Steps>
  <Step title="Open the Developer Portal">
    Go to [portal.alterai.dev](https://portal.alterai.dev) and navigate to your app.
  </Step>

  <Step title="Add Incident.io">
    Go to **Managed Secrets** > **Add Provider** > **Incident.io**.
  </Step>

  <Step title="Enter your credentials">
    Paste your API Key into the credential field.
  </Step>

  <Step title="Save">
    Click **Save**. You'll receive a `connection_id` to use with the SDK.
  </Step>
</Steps>

## Using in Code

```python
response = await vault.request(
    "YOUR_CONNECTION_ID",
    HttpMethod.GET,
    "https://api.incident.io/v2/incidents",
)
```
